===============================================
           코드 수정 프로세스 메모
===============================================

**토큰 사용량 최소화 규칙:**
- 모든 답변은 간결하게 (핵심만)
- 불필요한 설명 제거
- 한 번에 하나씩 작업
- 긴 설명 대신 요점만 제시

**코드 수정 5단계:**

0. 백업 확인
   - "백업 했나요?" 질문
   - "네" 답변 시 작업 진행

1. 파일 분석
   - 수정할 파일 읽기
   - 관련 파일 확인
   - 구조 파악

2. 수정 계획
   - 변경할 부분 명시
   - 이유 설명

3. 실행
   - 한 번에 하나씩 수정
   - 오류 시 중단

4. 검증
   - 린트 검사
   - 컴파일 확인
   - 기능 테스트

5. 요약
   - 변경 내용 정리
   - 영향도 설명

**Unity 적용:**
- 스크립트 컴파일 확인
- GameObject에 연결
- Inspector 설정
- Play Mode 테스트
